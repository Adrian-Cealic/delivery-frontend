Cum implementăm redesign-ul?

3 abordări tehnice — fiecare cu trade-off-uri diferite

A

Tailwind + shadcn/ui complet Recomandat

Instalăm Tailwind CSS + shadcn/ui. Ștergem CSS-ul custom, rescriuem toate componentele cu shadcn (Button, Card, Badge, Table, Dialog, Input, Select, Tabs). Lucide-react pentru iconuri. Rezultat: codebase modern, consistent, ușor de extins.

Avantaje

  • Design system complet și consistent
  • Dark mode built-in prin CSS variables
  • Componente accesibile (ARIA)
  • Exact stilul Vercel/Linear

Dezavantaje

  • Rescrierea completă a CSS + JSX
  • Dependențe noi (tailwind, radix-ui)
B

shadcn design tokens + CSS minim

Copiem design tokens-urile shadcn (CSS variables pentru culori, spacing, radius) în index.css fără a instala Tailwind. Rescriuem componentele cu clase CSS custom dar aliniate la sistemul shadcn. Fără dependențe noi majore.

Avantaje

  • Fără Tailwind — bundle mai mic
  • Aspect identic cu shadcn dark
  • Migrare mai rapidă

Dezavantaje

  • Nu poți folosi componente shadcn direct
  • Mai mult CSS manual de scris
C

Hybrid — shadcn pentru componente cheie

Tailwind + shadcn doar pentru componentele vizibile (Button, Badge, Card, Table, Dialog). CSS custom păstrat pentru layout, sidebar, stepper. Migrare treptată.

Avantaje

  • Risc scăzut — modificare incrementală
  • Păstrează ce funcționează deja

Dezavantaje

  • Design inconsistent (mix de stiluri)
  • Două sisteme CSS în paralel
  • Mai greu de întreținut

Recomandarea mea: A — Tailwind + shadcn complet. Rezultat cel mai curat și cel mai aproape de stilul Vercel.
